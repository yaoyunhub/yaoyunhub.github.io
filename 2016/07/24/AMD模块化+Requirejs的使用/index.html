AMD模块化+Requirejs的使用

前端模块化

模块化

说的就是对于js代码的管理！ 将指定功能的内容划分为一个个的模块，这些模块，在使用的时候可以被引用进来！有复用性，并且代码分门别类，容易维护和管理！！

CommonJS 模块化标准 代表就是Node.js (暂不讨论)

CommonJS ，主要用在Node.js当中

AMD 模块化标准

Async Module Definition 适用于前端开发
代表：Require.js，它是帮我们实现前端模块化的工具，他遵循的标准AMD。
所有的模块尽可能早的会被加载进来！！

定义模块
requirejs提供了一个全局的函数，define(),来定义模块

一个模块通常都会单独保存成一个文件
1 最基本的形式
define(funciton(){ /* 模块自己的代码 */ })
2 当模块要提供内容给别人使用的时候，需要使用return语句将内容进行返回
define(funciton(){ /* 模块自己的代码 */ return 定义的模块产出的内容 })
3 当模块依赖于别的模块的时候，需要给当前模块添加依赖项
依赖项如果有返回值，则需要一个形参来接收，下面就可以直接使用
define(["依赖项", "依赖项"], function(依赖项返回值){ /*具体执行代码*/ return 产出的内容 })

引用模块
requirejs提供了一个函数。来引用模块

require(["模块的名称（不需要.js后缀）"],callback)
第二参数是函数: 模块加载完毕之后的回调函数

+ 如果引用的模块有产出内容，在引用的模块内通过return返回内容，
+ 我们可以通过在回调函数中添加形参，来接收前面的模块的返回内容
+ 形参和模块是一一对应的

一般情况，会将有返回值的模块，全部放在第一个数组参数里面的前面！ 因为这样可以省略掉后面没有返回值的模块的形参

requirejs中，data-main属性的说明

  • 在请求requirejs的script标签中添加一个data-main属性，指向一个js文件
  • 当requirejs加载完成之后，会直接去执行这个data-maind属性所指向的js文件的内容

  • data-main所指的文件是异步加载的！所以如果在下面有依赖于data-main文件中实现的功能的代码，那么，很可能出错

    模块路径的研究

    1. 当直接使用require来引用一个模块的时候，那么模块的路径会以当前调用require方法的html页面为基础来查找
    2. 如果使用data-main属性，那么路径的查找，会以这个data-main所指的文件为基础来查找！
    3. 如果通过config方法指定了 baseUrl 那么所有的模块查找都会以baseUrl中设置的路径为基础

    注意：

    在设置了baseUrl之后，文件会首先按照baseUrl作为基础去查找，如果找到了，就直接使用，如果没有找到，按照 引用模块的文件本身作为基础再去查找！
    模块的应用路径可以写两种（都有效），一种按照baseUrl作为基础来写，另外一种就是按照当前文件路径来写

    require(["./modules/moduleA"]);
